私はいま、「Meguri（めぐり）」という名前の仮想的な通貨の構想を考えています。 仮想通貨 Meguri は従来の“所有するお金”ではなく、「循環そのもの」を本質とした存在としてデザインしたいと思っています。
## これを見ているみなさんへ

「こんな通貨だったら面白い」「こういう抜け道がありそう」「こんな世界観が合いそう」といった自由な発想を、ぜひ共有していただけたらすごく嬉しいです‥！ 世界観、比喩、物語、ゲーム的ルール、イラスト的イメージ、応援メッセージだけなど、どんな形でも大歓迎です。 みなさまの創造力でこの生き物が生まれるのを手伝ってほしいです。

基本的なイメージは次のとおりです。
• Meguri は“生き物のような性質”を持つ。 
• 時間とともに少しずつ自然に減っていく（富の偏りをやわらげるため）。 
• ただし社会の中でよく循環すると、Meguri 全体の総量がじわっと増える。
• 個人の残高が直接増えるのではなく、社会全体が豊かになる仕組み。
• 人が持つ「巣(ウォレット)」は、溜め込む金庫ではなく、Meguri が一時的に立ち寄る“巣”や“休憩所”のようなもの。 

いま特に考えたいテーマは次のような点です。
• 「巣（ウォレット）」はどんな場所であるべきか
• Meguri が好む“良い循環”とはどんなものか 
• 誰が、どんな形で社会全体の成長の恩恵を受け取るか 
• Meguri を「個別の生き物の群れ」として考えるか、「形のない流れ」として考えるか

# 巣
財布や口座ではない。

Meguriの「巣」は、所有の拠点ではなく、通過点として設計される。
“流れを可視化する場所”

だから、長くとどまるほど居心地が悪くなるようにできる。たとえば、動かさない時間が長いほど減価が強くなる。巣に籠もるほど、Meguriが弱っていく感じ。

逆に、よく出入りがある巣は“活気がある”。
頻繁にMeguriが出入りしている休憩所ほど、全体の生き物（Meguri総量）が元気になる方向に寄与する。巣そのものが評価されるというより、「そこを通る流れ」が評価される。

## Meguri における Sybil 問題まとめ

Meguri では、「巣（ウォレット）」を人や主体の単位として扱う。
このとき最大の問題になるのが Sybil 攻撃、つまり「1人が複数の巣を作り、別人のふりをして有利に振る舞う問題」。
通常の仮想通貨なら、KYC（本人確認）、生体認証、中央集権的なID管理などで防ぐが、Meguri の世界観ではこれらを極力使いたくない。
なぜならこれらは以下の理由により、Meguri の世界観と矛盾するからである。
- 管理者が「人間を管理する」構造になってしまうこと
- 所有・支配・監視の構図に戻ってしまうこと
- 「循環する生き物としての通貨」という思想と矛盾すること

つまり Meguri の Sybil 問題は、単なる技術的な認証問題ではなく、世界観そのものの問題でもある。

そこで Meguri では、Sybil を「排除」するのではなく、「意味化」するという発想を取る。
本人確認で潰すのではなく、不自然な存在のしかたを“育たない”構造にする。
つまり「1人 = 1巣」を強制するのではなく、複数の巣を自然に維持すること自体が極端に難しい世界を作る。

#### そのために考えている評価軸が、距離・時間・予測困難性（エントロピー）の三軸。
- 距離の軸では、巣の移動履歴を地図上で評価する
これにより不自然に遠距離を頻繁に移動する巣はスコアが育ちにくくなり、一度通った経路は価値が減衰する。
つまり、多数の巣を管理するコストが爆発的に増える
- 時間の軸では、滞在時間の分布、活動時間帯の偏り、リズムの自然さを評価する
これにより、人間1人が複数の生活リズムを同時に維持することを困難にする。
- 予測困難性の軸では、行動の確率分布からエントロピーを算出して評価する
機械的・定型的な行動は低エントロピー、生物的・即興的行動は高エントロピー
これにより、Bot的・組織的な巣の大量生成を弱体化させる。

この設計の核心は、「何をしたか」ではなく「どう存在しているか」を評価する点にある。
従来の通貨や報酬設計は「何をしたか → 報酬」という構造だが、
Meguri では「どう存在しているか → 育ちやすさ」となる。
たくさん持つ、たくさん動かす、たくさん稼ぐのではなく、
自然に生きているか、偏らず循環しているか、無理のないリズムを持っているかがスコアを育てる。
この構造では、Sybil 攻撃は「構造的に割に合わない行為」になる。

さらにこの世界では、巣をたくさん持つこと自体が、行動エントロピーを下げ、時間資源を奪い、結果的に全巣のスコアを劣化させる。
つまり攻撃が自己弱体化に変換される。
この設計は Proof of Work や Proof of Stake とは異なる、「Proof of Being（存在証明）」とも言える新しいカテゴリに近い。

現在考えている数式モデルは、
巣の評価 = ( Σ w(dᵢ) ) × ( ∫ s(t) dt ) × ( − Σ pᵢ log pᵢ ) という形で、
距離コスト、時間蓄積、行動エントロピーを掛け合わせることで、
自然に存在するほど増え、不自然に振る舞うほど減る性質を持たせようとしている。
この数式は Sybil 対策であると同時に、Meguri の世界観そのものを駆動するエンジンでもある。
