全6討論を横断した総合指摘

Phase 2 全体を通読した上での、討論間の矛盾・設計の穴に対する4つの根本的指摘。

1
Phase 2-03 雨モデル × Phase 2-05 二層構造
雨モデル（均等配分）と品質評価（Layer 2）のインセンティブ崩壊
未解決
雨モデルで全員に1/N均等配分するなら、Layer 2 の softmin スコアが高くても何の得もない。じゃあ誰がわざわざ「良い循環」をするの？ インセンティブが完全に崩壊してるじゃん。品質評価やる意味ある？

分析

これは設計上の本質的な穴。Phase 2-03 で「雨モデル = 全巣に1/N均等配分」を導入し Sybil ROI をゼロにしたが、Phase 2-05 で導入した softmin スコア（Layer 2）の「使い道」が不明確なまま残っている。

  • Layer 1（異常検知）= バイナリ判定 → Mana 配分の参加資格として機能
  • Layer 2（softmin スコア）= 連続値 → 配分額に影響しないなら存在意義が不明
  • 「評価するが報酬に反映しない」は理念としては美しいが、実際の行動動機として機能しない可能性
Phase 2-03 雨モデル → Phase 2-05 二層構造 →
2
Phase 2-05 AI受容 × Phase 2-03 雨モデル
AI 大量参入による人間の Mana 希釈問題
未解決
「AIも存在するなら受け入れる」って言ったよね？ で、雨モデルは1/N均等配分。AI が巣を100万個作ったら人間の取り分は100万分の1に希釈される。「識別しない」と「均等配分」を同時にやったら、人間が飢え死にするだけじゃん。

分析

Phase 2-05 の「AI 受容」と Phase 2-03 の「雨モデル（1/N配分）」の組み合わせが生む構造的矛盾。

  • Sybil 対策（人間が複数ウォレットを作る）には雨モデルが有効 → ROI ゼロ
  • しかし自律的 AI が独立した存在として大量参加するケースは Sybil とは別問題
  • Layer 1 の異常検知で AI を弾くなら「AI 受容」と矛盾
  • 弾かないなら N が爆発的に増加し、人間1人あたりの Mana が生活不能水準まで希釈
  • 「AI 受容」の範囲と条件の明確化が必要

3
Phase 2-06 静寂閾値 × Phase 2-01 減衰加速
静寂閾値 + 減衰加速 = デジタル追放（復帰不能）
半解決
90日動かなかったら分母Nから除外 + 減衰加速。病気や災害で長期離脱した人は通貨がゼロ近くまで減って、復帰しても取り戻す手段がない。それって事実上のデジタル追放でしょ。「自然に存在するものを守る」って言ってなかった？

分析

Phase 2-05 で「猶予期間（grace period）」が導入済みだが、これは新規参入者・移動者向け。長期離脱からの復帰シナリオが明確にカバーされていない。

  • 新規参入者 → grace period あり（Phase 2-05 で合意済み）
  • 引っ越し → 移動先での grace period あり（Phase 2-04 で合意済み）
  • 病気・災害での長期離脱 → 復帰パスが未定義
  • 巣の相続は却下済み（タイムラグ攻撃の脆弱性 / Phase 2-06）
  • 「復帰用 grace period」の設計が必要（新規と同等の扱い？ それとも別枠？）

Phase 2-04 距離軸廃止 × Phase 2-02 循環品質
距離軸廃止後のコミュニティ境界定義の曖昧さ
半解決
GPS距離軸を廃止したのはいい。でもコミュニティ相対評価を導入するなら、「コミュニティの境界」をどう決めるの？ 結局位置情報が必要になるか、取引グラフのクラスタリングが必要になるけど、Phase 2-06 でクラスタリングはスコアリングから除外したよね？

分析

Phase 2-04 で「取引グラフのネットワーク距離でコミュニティを定義」と概念的に合意済み。しかし Phase 2-06 で「クラスタリングはスコアリングから除外（研究用のみ）」とした結果、実装パスに矛盾が生じている。

  • 概念: コミュニティ = 取引の密なクラスター（GPS不要）
  • 問題: クラスタリング結果を Q_equity 正規化にどう使うかが未定義
  • 「スコアリングに使わないが正規化には使う」は論理的に整合するが、具体的な実装設計が必要
